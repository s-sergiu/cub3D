Detailed Description

A cross-platform OpenGL graphics library based on the idea that MiniLibX provides. Just quite a bit better, in terms of code quality & performance.

As for the few void* present in some structs and functions and why MLX is split into two different headers, so to speak, it's mainly for abstraction. Most users won't have a need for the inner workings of MLX (shaders, ...) and it also helps keep MLX nice and tidy. Base object for disk loaded textures. It contains rudementary information about the texture.

Parameters
widthThe width of the texture.
heightThe height of the texture.
pixelsThe literal pixel data.
bytes_per_pixelThe amount of bytes in a pixel, always 4.
