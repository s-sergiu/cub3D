\doxysubsection{Detailed Description}
A cross-\/platform Open\+GL graphics library based on the idea that Mini\+LibX provides. Just quite a bit better, in terms of code quality \& performance.

As for the few void$\ast$ present in some structs and functions and why MLX is split into two different headers, so to speak, it\textquotesingle{}s mainly for abstraction. Most users won\textquotesingle{}t have a need for the inner workings of MLX (shaders, ...) and it also helps keep MLX nice and tidy. Base object for disk loaded textures. It contains rudementary information about the texture.
